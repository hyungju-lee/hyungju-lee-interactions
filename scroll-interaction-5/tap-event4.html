아, 당신은 CORS 제한에 도달했습니다.
그리고 여기에서 가장 공격적인 구현 자로 악명 높은 Google 크롬에서이 문제가 발생하고 있다고 생각합니다.
나는 이것을 많이 보았다.

CORS는 교차 출처 콘텐츠가 웹 페이지에 삽입되는 것을 방지하는 프로토콜입니다.
이는 스크립트 파일에 영향을 미칠뿐만 아니라 (예상대로 웹 페이지에 악의적 인 스크립트를 삽입하는 것을 원하지 않기 때문에) 이미지 및 글꼴과 같은 리소스 에도 영향을줍니다.

이미지에 영향을 미치는 이유는 악의적 인 개인이 HTML5 캔버스 개체를 사용하여 웹 페이지의 콘텐츠를 PNG 파일로 복사하고 원하는대로 개인 데이터를 가져갈 수 있음을 발견했기 때문입니다. 이것이 당신에게 일어난 동안 인터넷 뱅킹 거래에 참여!

그러나 이것은 여러분이 직면하고있는 성가신 부분입니다.
이러한 악의적 인 활동을 중지하면 교차 출처 리소스의 합법적 인 사용 에도 영향을줍니다 (예 : 모든 이미지를 별도의 저장소에 보관).

그래서이 문제를 어떻게 해결합니까?

Firefox에서는 문제가 없습니다.
Firefox는 문제에 약간의 지능을 적용하고 웹 페이지와 동일한 file : // 사양에서 오는 이미지가 실제로 교차 출처가 아니라는 것을 인식합니다.
웹 페이지와 하드 드라이브의 동일한 디렉토리에있는 한이를 통과 할 수 있습니다.

반면에 Chrome은 훨씬 덜 관대합니다.
이러한 모든 액세스를 교차 출처로 취급 하고 캔버스에서 getImageData () 및 putImageData ()를 사용하려고하는 순간 보안 종료를 구현합니다.

자체 로컬 웹 서버를 설치하고 구성하는 문제를 겪고 싶지 않지만 여전히 Chrome 및 친절하고 친숙한 디버거를 사용하려는 경우 해결 방법이 있습니다.
Chrome 실행 파일을 가리키고 두 번 클릭하면 실행되는 바로 가기를 만들어야하지만 특수 명령 줄 플래그를 사용하여 Chrome이 시작됩니다.
